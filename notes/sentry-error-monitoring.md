# Sentry error monitoring

Sentry is a developer platform for error monitoring, debugging, and distributed tracing.

A German Sentry landing page describes it as "Fehlerüberwachung" and includes the copyright notice "© 2026 • Sentry ist eine eingetragene Marke der Functional Software, Inc.".

Sentry's docs page says it provides "end-to-end distributed tracing" to help developers identify and debug performance issues and errors across their systems and services.

A Sentry docs page for AI says it can connect to AI coding assistants and workflows, including agent skills, the MCP server, and monitoring for AI applications.

Another capture quotes Sentry AI copy: "agent skills, use the MCP server, or add monitoring to your AI applications" and says agent skills teach an AI coding assistant how to set up and configure Sentry, working with Claude Code, Cursor, GitHub Copilot, and more.

A newer capture introduces "dotagents" as "A package manager for .agents directories" that lets teams declare agent skill dependencies in `agents.toml`, lock versions for reproducibility, and share skills across the team.

A later capture expands on dotagents: without a package manager, agent skills get copy-pasted between directories, versions drift across team members, and onboarding new tools requires manual setup.

Another capture says dotagents can declare MCP servers once in `agents.toml`, then generate the correct config file for each agent during install.

A new capture says dotagents can also declare hooks once in `agents.toml` and write the correct hook config for each supported agent.
